Does gaming on Linux suck?

In the recent years, we have seen Linux mature into a stable and user-friendly operating system. Thanks to the countless hours spent by the developers, some Linux distributions are even more stable that Mac OS and Windows. Even though there is some learning curve for a beginners, but that stability, productivity, or even in some distributions, the bleeding edge features, is worth all that time you spend learning it. These are the things that attract more geeks to escape the windows or mac environment and use Linux as their primary OS.

As there are a lot of different distributions, we tend to believe that there is something for everyone. I have covered an article discussing, different Linux distributions and which one is best for you. But is that true for gamers? Let us find out!

Can you game on Linux?

There is no simple answer to this question. Yes you can game on Linux but not all the games and there are couple of things that you have to keep in mind if you plan to game on Linux
1) Most of the Distributions except Ubuntu and steam OS have no official support for steam. But you can install Steam through wine. Wine if you don’t know, is a free and open source compatibility layer which helps you run the windows applications and games on Linux. I use it on Debian, and it works as intended for the most part, but there some crashes here and there.

2)No official support for the graphics card drivers from AMD and Nvidia is a sour point for the Linux. Hence you have to look for it on the web. If you have a graphics card that isn’t that popular, then good luck finding its drivers! The open source drivers are not in power with the performance of those launched officially. Hence you will definitely be noticing FPS drops and performance difference.

3) You can’t play all the games on Linux. Most of the AAA titles don’t run on the Linux. So using Linux as your primary OS for gaming will make you miss out on many great games.

4)The games with DX11 and DX12, don’t run out of the box. You have to spend a lot of time browsing through the forums in order to tweak the settings and make it run. But even if you make it run somehow, there will be lack of optimization and hence decrease in performance.

Gaming experience on Steam OS and Ubuntu

Ubuntu and steam OS officially support Steam. They both are based on Debian. You guys might be families with Ubuntu, most of us have used it as out first Distribution. Let me tell you about SteamOS. It is an open source distribution put in together by Valve for the so called “living room entertainment environment”. It aims towards making it more user friendly so that even a not technical gamer can use it. There has been a significant increase in optimization and performance, that been said, we still have still have a long way to go. In the current development stage, it is not advised for the non technical person (as stated on the steamOS website.)

My gaming background

I use Debian as my primary OS for programming . I game for around 2- 3 hours at night. Now the thing is, at the end of the day, I am so tired that I just need the game to run out of the box. I don’t want it to crash while I am playing. That’s the reason why I use windows. Plus I am very picky about the performance af the laptop on which I game sucks, hence I need to grasp every bit of performance.

My views

I might have seemed hard on Linux, making you feel like I hate it, but that's not true. I love the flexibility and customization that Linux brings with it. But the first step to solve the problem is accepting there is one. Only then we can look forward addressing it. Now that we have Ubuntu and Valve working on making the performance better, the day is not far, when we will be witnessing Steam OS as a major platform for gaming.
But looking at this development stage, I wouldn’t recommend using it as your primary OS for gaming. If you are curious enough, then you can try running few games on Linux.
