Secure Web Browsing -Part 1 ( Introduction to HTTP and HTTPS)


While browsing the internet, you guys might have noticed that some sites using HTTP, while the others, using HTTPS. Which one of them is secure? And does using any of them enhances the browsing experience? Lets find out!

By now you might have known that HTTPS is secure probably because popular sites like Twitter and Facebook use it, and probably because it has that padlock symbol in the address bar.

What is HTTP?

HTTP stands for Hypertext Transfer Protocol.HTTP is an application protocol that runs on the top of the TCP/IP protocol. In simple words, this is what helps you browse the Internet. When you search something on the web, you are basically sending a request to the server, that sends you the packets over TCP/IP.

Webpage formation

When you request a server for a website, it is not only that server which is sending information to the web-page. There are a couple of different servers apart from that, from the different parts of the world that are sending you some information. Generally, that information is in the form of the adverts.

As you can see in the above picture, these different bits and pieces of the information from different servers that create the web page. There are some banners at top and button and some adverts at the side and the main content which actually wanted to see.

As the servers, from which the web page is grabbing data from, are located across the world, which makes the system more complex and there are potentially weak links, and if data from the server is redirected, it causes some serious damage.

If you have a slow internet connection, you might have noticed that as the page renders, the things tend to move a round a bit as the the site is getting data from the various servers across the planet.

Security flaws with HTTP

  • In HTTP, you have no idea if the request that you are sending is being sent to the right server or not. Your request can be redirected and sent to some other server. That server might send you a web page that contains millions of cookies that will track you on the internet for your rest of your life.

    Your ISP can redirect all your request. And it makes changes in the packet, as in this case, he removed one of an advert from the Google and replaced it with his ad.I can’t comment about the legality of the situation, but this surely is unethical and the person who has paid Google to show his ads is suffering a loss.

  • The packets that you receive from the server can be altered before it reaches you, hence you end up getting modified information. Sadly, there is no way the user can know if what he is looking at is authenticate.
  • Anyone in your network can perform man in the middle attack and can see what you are browsing as all the text is unecrypted
  • Making things worse, if you enter your account details in the webpage which is on HTTP, your credentials can be seen by the attackers doing man in the middle attack, and even by your ISP as all the traffic fist pass through him.

    HTTPS – covering up the flaws of HTTP

    HTTPS aims towards covering up the flaws of HTTP. Seeing its increasing popularity, we can definitely assume, it has successfully done it. Here are some of its features -

    • HTTPS encrypts all the user data using public key encryption, which by the way you can read more by clicking here. It basically assures that the request you are sending is going to the right server and the packets you are receiving are not being altered.

    • If an attacker launches a man in the middle attack, the data he is able to capture will be encrypted and it will all just look like the noise. Most of the banks’ website and even the popular sites like Twitter and Facebook use HTTPS, hence making it safe to input your credentials.

    The internet is slowly but steady adapting towards HTTPS, which is a step in right direction. With the use of public key encryption, I am expecting this protocol to last longer before someone finds a vulnerability. That being said, it’s not like HTTPS is completely secure.
