Generate wordlist using Crunch for password cracking.

Welcome back geeks, how you all doing?

In this post, I will be discussing crunch and how we can use it to generate a wordlist which we can further use to crack passwords. So without wasting much time, let us hop on to this geeky journey of understanding crunch.

What is crunch?

In a nutshell, crunch is a tool that generates all the possible permutations and combinations. When you enter a specific characters of password or even a pattern that you might know. It will also calculate the number of combinations that it will generate and then you can estimate the overall time it will be consuming based on your PC’s hardware.

You can then use the generated wordlist with tools like aircrack-ng, Rainbow Crack and john the ripper to launch a successful attack.

When and why should you use crunch?

In password cracking principles, we tend to prefer using the dictionary attack, where we essentially attempt thousands of passwords per second from a specified word list. A good wordlist contains all the common passwords that people use.

But if you live in a non-English-speaking country, then dictionary attack might not be helpful. You might find a wordlist, but it won’t be good enough. Hence crunch will be your best option.

Sometimes, you know the person whose password you want to crack. You might know password’s length or certain characters he has used or some pattern, then crunch is indeed your go-to guy.

Using crunch

Using it is quite simple, even though I will be showing you some examples for better clarification, but you can simply skimp through the man pages by typing-. man crunch It is quite descriptive and easy to understand.

Crunch syntax

crunch <min_characters > <max_characters> <characters> -p <pattern> -o <output_file>

OR

crunch <min_characters > <max_characters> <characters> -p <pattern> -o <output_file> |<pass_through_application>

  • min_characters– Minimum password length.
  • max_characters- Maximum password length.
  • characters- here you mention all the characters, that you think there are in password. It can be all the way from a-z and 0-9 and special characters like @, &, *, $, etc.
  • Pattern- If you think that the person uses his birthday date in the password or you know a position of specific characters in password, then you can even mention that. I will be showing you one of such example below.
  • Output_file- This is the file, where all the results will be saved.
  • pass_through_application- If you want to directly pass all the results in password crackers like aircrack-ng, then you can put it there.
