Beginners to advance guide for ICMP -Part 2 (ICMP message types)

Welcome back, fellow geeks! In my 1st part, I discussed about the basics of ICMP and then understood the structure of an ICMP error message. In this part, I will be discussing about the the different ICMP message type.
If you have stumbled across this article and have no clue what ICMP is, then go check out the 1st part.

In case you have forgotten…..

The error messages are divided into different sections called types and to give in depth information about the error, these types have their own section called code.

Different types of ICMP messages

Now that we are clear with the basics of ICMP and the structure of the ICMP packages, lets talk about the different ICMP messages types. Remember, I mentioned about the so called “technical issues ”, this is the part I was talking about. One thing to note is that, I am not writing all the ‘codes’ of every'type’. I will be just discussing the popular ones.

ICMP Destination Unreachable (Type 3 )

The first three fields seen in the image are the same as previously described. The Destination Unreachable type has 6 basic codes that can be used, as seen below in the list.
  • Network unreachable (Code 0) - Tells you if a specific network is currently unreachable.
  • Host unreachable (Code 1) - Tells you if a specific host is currently unreachable.
  • Protocol unreachable (Code 2) - This code tells you if a specific protocol (tcp, udp, etc) can not be reached at the moment.
  • Port unreachable (Code 3) - If a port (ssh, http, ftp-data, etc) is not reachable, you will get this message.
  • Fragmentation needed and DF set (Code 4) -If a packet needs to be fragmented to be delivered, but the Do not fragment bit is set in the packet, the gateway will return this message.
  • Source route failed(Code 5) -If a source route failed for some reason, this message is returned.
  • Destination network unknown(Code 6 ) -If there is no route to a specific network, this message is returned.
  • Destination host unknown (Code 7 ) -If there is no route to a specific host, this message is returned.

Source Quench (Type 4)

If the receiver feels that there are a lot messages he is receiving, then he will send a ICMP error message to send the package slowly. It also informs that the datagram is discarded due to a lot of data. I source quench error message is sent with 1 datagram discarded. All source quench packets have their ICMP types set to 4. They have no codes except 0.

Time to live (TTL) equals 0 (Type 11 )

As discussed earlier, TTL is used to prevent looping of a packet in network. I has some set value which determines how many routers you can cross (hop). After every hop, its value is decremented by 1. If in case the package goes into a loop, its hop value will eventually decrease and would finally die. (We are gonna be talking more about TTL in Traceroute command)

ICMP Echo Request (type 8)/Reply(type 0.)

Echo request and reply are related to each other. The first difference is that the echo request is type 8, while echo reply is type 0. When a host receives a type 8, it replies with a type 0. When the reply is sent, the source and destination addresses switch places as well. After both of those changes has been done, the checksum is recomputed, and the reply is sent. There are only 1 code for both of these types, they are always set to 0. (We are gonna be practically applying echo request/reply in ping command)

Redirect (type 5)

The ICMP Redirect message is used to notify a remote host to send data packets on an alternative route. A host should not send an ICMP Redirect message. Redirects should only be sent by gateways.
  • Redirect for network (Code 0) - Only used for redirects for a whole network (e.g., the example above).
  • Redirect for host (Code 1) - Only used for redirects of a specific host (e.g., a host route).

I haven’t covered all the types and codes, just mentioned those which are important and popular. If you like to know every ICMP message, then there is a nice descriptive table on wikipedia, check that out.
