Getting acquainted with some words

These are not exactly new terms in the networking. These are basically some full forms.
ACK – Acknowledge
SYN – Synchronize
SEQ – Sequence

Getting a basic idea about three way handshake

Three way handshake is the first step taken by the client to connect to the server. In other words, this the first thing that happens, before the client sending and receiving to the server. Client: The client performs the active open and sends a SYN request to the server. Which basically means, hey server do you wanna talk?

Server: The server is like yes, I am okay with you talking, but do you want to talk to me?
Hence it then sends an SYN and ACK as a gesture that it has received client’s SYN request.

Client: It then says, Yes I want to talk to you. Lets talk!
It then again sends an ACK to the server.
And now the connection is established and they can send data through TCP.

Digging deep into three way handshake

Now that we are clear with the basic concept of three way handshake, let us dig deep into it and talk numbers here. This is the same image that I have posted above, but it is a bit more detailed. Even though the image is the best representation of the three way handshake, but if you didn’t understand from the image, I have explained it below. So here the client sends the SYN request to the server which has the SEQ = 200 . The server acknowledges it by sending ACK which is +1 of the SEQ sent by the client (ACK = SEQ+1, henceforth ACK = 200+1)and its own SEQ number, which in this case is 100.
Then the client acknowledges it by sending ACK which is +1 of SEQ number, that was sent by the server(ACK = SEQ + 1, henceforth ACK= 300+1).

Answering common question....

Question –How are the numbers, 200 and 300 chosen?
Answer -The first SEQ number sent by the client (in this case it was 200) or the SE sent by the server to the client (in the above case, 100 )in the second stage are randomly chosen numbers. There are no specific numbers.

Question –Will the ACK always be increased by 1?
Answer –yes
